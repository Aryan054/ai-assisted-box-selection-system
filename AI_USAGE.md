# AI Usage Report

## Project

**AI-Assisted Box Selection System**

---

## Purpose

AI tools were used as a development assistant to improve productivity, understand best practices, and assist with code organization. All generated code was reviewed, modified where necessary, integrated into the project, and tested before submission.

---

## AI Tool Used

- ChatGPT (OpenAI)

---

## Areas Where AI Was Used

### 1. Project Planning

AI was used to:

- Break down the assignment into manageable tasks.
- Suggest a clean Django project structure.
- Recommend a layered architecture (Models → Serializers → Services → Views → URLs → Tests).

---

### 2. Database Design

AI assisted in designing the database models for:

- Product
- Box
- Order
- OrderItem

The generated model suggestions were reviewed and adapted to meet the assignment requirements.

---

### 3. Django REST Framework APIs

AI helped generate the initial structure for:

- CRUD APIs
- Class-Based Views
- URL routing
- Serializers

The implementation was manually verified and integrated into the project.

---

### 4. Recommendation Algorithm

AI assisted in designing the box recommendation logic based on the assignment rules:

- Product dimension validation
- Weight validation
- Product rotation
- Lowest cost selection
- Smallest volume tie-breaker
- Alphabetical name tie-breaker

The algorithm was manually reviewed, tested, and refined.

---

### 5. Code Refactoring

AI provided suggestions for improving:

- Code readability
- Method extraction
- Separation of concerns
- Service layer implementation
- Naming conventions

---

### 6. Unit Testing

AI was used to generate ideas for:

- API test cases
- Service layer tests
- Edge case testing
- Validation scenarios

All tests were executed locally before submission.

---

### 7. Documentation

AI assisted in drafting:

- README.md
- API_DOCUMENTATION.md
- AI_USAGE.md

The final documentation was edited and verified manually.

---

## Manual Work Performed

The following tasks were completed manually:

- Project setup
- Django configuration
- Database migrations
- Code integration
- Debugging
- API testing
- Unit testing
- Validation of recommendation logic
- Running Black
- Running Flake8
- Running Coverage
- Git version control
- Final verification

---

## Verification

Every AI-generated suggestion was:

- Reviewed before use
- Modified where necessary
- Executed locally
- Tested using Django's test framework
- Verified against the assignment requirements

No code was copied directly into the final submission without review and testing.

---

## Summary

AI was used as a development assistant to improve productivity and code quality. Final implementation decisions, testing, debugging, and verification were performed manually to ensure the application satisfies the assignment requirements.
